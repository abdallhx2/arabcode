---

## 1. نبذة عامة

OpenCode أداة برمجة بالذكاء الاصطناعي **مفتوحة المصدر بالكامل**، متوفرة بأربع واجهات: طرفية (TUI/CLI)، تطبيق سطح مكتب (Desktop)، إضافة داخل محرر الأكواد (IDE Extension)، وواجهة ويب. الشركة المطوّرة هي Anomaly (anoma.ly).

**الفرق الجوهري عن Claude Code:** OpenCode مصمم من الأساس ليكون *provider-agnostic* — غير مرتبط بمزود موديلات واحد.

## 4. الميزات الأساسية — بالتفصيل

### 4.1 تعدد مزودي الموديلات (Multi-Provider Support)
**الشرح:** OpenCode مبني حول قائمة مزودين ضخمة عبر Models.dev — يدعم OpenAI، Anthropic Claude، Google Gemini، AWS Bedrock، Groq، Azure OpenAI، OpenRouter، GitHub Copilot، Cohere North، MiniMax، GLM، DigitalOcean، Snowflake Cortex، وأكثر. كما يدعم موديلات محلية self-hosted (OpenAI-compatible) عبر متغير `LOCAL_ENDPOINT`.
**الفائدة:** تقدر تبدّل الموديل حسب المهمة (موديل رخيص للمهام البسيطة، موديل قوي للتعقيد) بدون تغيير الأداة، وتربط موديلاتك المحلية (زي اللي على DGX Spark) بنفس الواجهة.

### 4.2 أوضاع Build / Plan
**الشرح:** وكيلان مدمجان تتنقل بينهما بمفتاح Tab: `build` (وصول كامل للتعديل والتنفيذ) و`plan` (قراءة فقط، يقترح خطة بدون لمس الملفات).
**الفائدة:** مراجعة الخطة قبل التنفيذ تقلل من التعديلات الخاطئة على كود حساس أو إنتاجي.

### 4.3 الوكلاء المخصصة (Custom Agents)
**الشرح:** إنشاء وكلاء عبر `opencode agent create` بصلاحيات دقيقة (bash, read, edit, glob, grep, webfetch, task, websearch...). كل وكيل يُعرّف بملف Markdown يحوي YAML frontmatter للإعدادات وجسم النص كـ system prompt. الوكلاء إما `primary` (يقود المحادثة) أو `subagent` (يُستدعى للمهام المتخصصة).
**الفائدة:** فريق من الوكلاء المتخصصين (مراجع أمان، كاتب توثيق، خبير اختبارات) بدل وكيل عام واحد يعطي نتائج أدق لكل مجال.

### 4.4 AGENTS.md
**الشرح:** مكافئ CLAUDE.md — أمر `/init` يحلل المشروع وينشئ ملف AGENTS.md بالجذر يوثق التقنيات، أسلوب الكود، البنية، وأوامر التطوير الشائعة. يُقرأ تلقائيًا كل جلسة.
**الفائدة:** لا تحتاج تذكّر الوكيل بقواعد مشروعك كل مرة — يتوافق تعامله مع الكود مع اصطلاحاتك مباشرة.

### 4.5 Session Snapshots والتراجع (v1.17.11)
**الشرح:** حفظ لقطة لكل جلسة تسمح بالرجوع لأي رسالة سابقة، شامل استعادة التعديلات على الملفات وقت تلك اللحظة.
**الفائدة:** لو أخذ الوكيل مسارًا خاطئًا في تعديل معقد، ترجع لنقطة محددة بدل التراجع خطوة بخطوة أو فقدان العمل بالكامل.

### 4.6 دعم MCP (Model Context Protocol)
**الشرح:** إدارة كاملة لخوادم MCP — إضافة محلي أو بعيد، عرض حالة الاتصال، مصادقة OAuth، قوالب موارد (resource templates)، وأدوات قراءة موارد. إضافة حديثة مهمة: **Code Mode MCP Adapter** (v1.17.14) يسمح بتشغيل سكريبتات orchestration مقيّدة فوق أدوات MCP المتصلة بدل استدعائها واحدة تلو الأخرى.
**الفائدة:** تكامل مباشر مع أدوات خارجية (قواعد بيانات، APIs، أنظمة الملفات) وتنسيق منطق معقد بينها بكفاءة أعلى.

### 4.7 YOLO Mode (v1.17.12)
**الشرح:** وضع بالـ TUI يوافق تلقائيًا على كل طلبات الصلاحيات بدون مقاطعتك في كل خطوة.
**الفائدة:** سرعة أكبر بالمهام الطويلة الموثوقة على حساب طبقة الأمان اليدوية — يُستخدم بحذر في بيئات تثق فيها بالكود.

### 4.8 الأوامر المخصصة (Custom Commands)
**الشرح:** ملفات Markdown في مجلدات محددة تصبح أوامر جاهزة، تدعم متغيرات مسمّاة بصيغة `$NAME` يُطلب منك تعبئتها عند التشغيل.
**الفائدة:** تحويل مهام متكررة (فحص PR، تجميع سياق issue) إلى أمر واحد بدل كتابة نفس التعليمات كل مرة.

### 4.9 أوضاع التشغيل غير التفاعلية (للأتمتة)
- **`opencode run "..."`**: تنفيذ غير تفاعلي مباشر، مناسب للسكريبتات، يدعم إخراج JSON خام و`--replay` لإعادة تشغيل الجلسة.
- **`opencode serve`**: خادم HTTP بدون واجهة TUI للوصول البرمجي الكامل.
- **`opencode web`**: نفس الخادم مع واجهة ويب تُفتح تلقائيًا.
- **`opencode --mini`**: وضع CLI مصغّر (v1.17.10).
- **`attach`**: ربط طرفية TUI بخادم backend شغّال عن بُعد.
- **خادم ACP**: تواصل عبر stdin/stdout بصيغة nd-JSON لتكاملات الأدوات المتقدمة.

**الفائدة:** تغطية كاملة لسيناريوهات الأتمتة — من سكريبت bash بسيط إلى خادم API إنتاجي، مع تفادي "cold boot" لخوادم MCP عبر الاتصال بجلسة `serve` قائمة بدل تشغيل جديد كل مرة.

### 4.10 تكامل GitHub/GitLab
**الشرح:** `opencode github install` يهيّئ GitHub Actions workflow تلقائيًا، و`opencode github run` يشغّل الوكيل داخل CI — مفيد لمراجعات PR آلية. `opencode pr <رقم>` يجيب فرع PR من GitHub ويشغّل عليه مباشرة. بالإضافة، **GitHub أعلنت رسميًا دعم Copilot Pro/Business/Enterprise كمزود مصادقة مباشر داخل OpenCode**.
**الفائدة:** مراجعة كود آلية داخل خط CI/CD، واستخدام اشتراك Copilot الموجود عندك بدل مفتاح API منفصل.

### 4.11 LSP Integration
**الشرح:** دعم Language Server Protocol لذكاء برمجي حقيقي (اقتراحات دقيقة، معرفة أنواع، تنقّل بالكود) وليس فقط تحليل نصي.
**الفائدة:** فهم أعمق لبنية الكود يقلل من الأخطاء الناتجة عن تخمين النوع أو المرجع.

### 4.12 إدارة الجلسات والتكلفة
**الشرح:** عرض/حذف/تصدير الجلسات (JSON)، نقل جلسة بين مجلدات (v1.16.0)، استنساخ workspace كامل مع الملفات غير المرفوعة لـ git، أمر `stats` لعرض استهلاك التوكنز والتكلفة، وأمر `/share` لإنشاء رابط مشاركة.
**الفائدة:** شفافية كاملة على التكلفة والاستخدام — مهم لو تدير عدة مشاريع أو عملاء وتحتاج تتبع الإنفاق لكل واحد.

### 4.13 نظام Skills
**الشرح:** اكتشاف تلقائي للـ skills ووكلاء مبنية على ملفات (v1.16.0)، مع مسارات مخزّنة كمسارات ملفات حقيقية بدل `file://` URLs (v1.17.10).
**الفائدة:** إعادة استخدام منطق متخصص محفوظ مسبقًا بدل إعادة شرحه كل جلسة.

### 4.14 التوافق مع Claude Code
**الشرح:** متغيرات بيئة مثل `OPENCODE_DISABLE_CLAUDE_CODE` و`OPENCODE_DISABLE_CLAUDE_CODE_SKILLS` تشير إلى أن OpenCode يقرأ مجلد `.claude` تلقائيًا (بما فيه CLAUDE.md والـ skills) إذا كان موجودًا بالمشروع.
**الفائدة:** لو عندك skills أو CLAUDE.md مبنية مسبقًا لـ Claude Code، تقدر تجربها في OpenCode بأقل تعديل.

### 4.15 Plugins وSDK
**الشرح:** نظام plugins قابل للتوسعة (hook APIs مسمّاة، V2 plugin API لـ Effect وPromise)، وSDK كامل يعرض: بث أحداث حيّ (live event stream)، وصول للجلسات النشطة، سجل جلسات مقسّم صفحات (paged durable history)، ونقاط نهاية لطلبات الصلاحيات.
**الفائدة:** بناء أدوات وتكاملات مخصصة فوق OpenCode كمنصة، وليس فقط استخدامه كتطبيق جاهز.

### 4.16 التثبيت والمنصات
**الشرح:** تثبيت عبر curl script، npm/Bun/pnpm/Yarn، Homebrew (macOS/Linux)، pacman/AUR (Arch)، Chocolatey/Scoop (Windows)، Docker، وmise. على Windows يُنصح بـ WSL للتوافق الكامل، وتم دعم WSL-backed Desktop رسميًا (v1.17.0) مع تحسينات إعداد لاحقة (v1.17.13).
**الفائدة:** مرونة تثبيت كاملة تناسب أي بيئة تطوير عندك، من Arch Linux إلى Windows عبر WSL.

## 5. خلاصة سريعة — الأقوى في OpenCode

1. **الاستقلالية عن المزود الواحد** — أكبر فرق عن Claude Code، مهم جدًا لو تبي تجرب موديلات محلية أو تقارن بينها.
2. **الأتمتة الشاملة** — أوضاع run/serve/web/ACP تغطي كل سيناريوهات السكريبتات والإنتاج.
3. **Session Snapshots** — تراجع دقيق لأي نقطة، مو بس undo خطوة بخطوة.
4. **التوافق مع بنية Claude Code** — إمكانية إعادة استخدام CLAUDE.md وskills موجودة عندك.
5. **وتيرة تحديث سريعة جدًا** — إصدار كل يومين تقريبًا (v1.17.7 إلى v1.17.16 خلال أقل من شهر)، يعني تطور نشط لكن يتطلب متابعة changelog بانتظام.
