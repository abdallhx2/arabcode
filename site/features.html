قوّة أداة برمجة حديثة — بالعربية الكاملة

arabcode مبني على محرّك opencode مفتوح المصدر، فيرث منظومة ميزات ناضجة ومختبَرة، ويضيف فوقها ما يميّزه فعليًا: عرضٌ صحيح للعربية داخل الطرفية.

عن النسب الميزات الموروثة هي من opencode؛ وإسهام arabcode الأساسي هو طبقة العربية (الاتجاه والتشكيل والإدخال) المطبّقة فعليًا في هذا المستودع. لا يدّعي arabcode اختراع الميزات الموروثة.

ميزة arabcode المميِّزة

١. دعم العربية الكامل (RTL + تشكيل)

هذه هي الإضافة الحقيقية التي يقدّمها arabcode فوق opencode، وهي سبب وجوده — تجربة عربية أصيلة في الطرفية تقرأ فيها ردود الوكيل وتكتب طلباتك بالعربية دون حروف مقطّعة ولا اتجاه معكوس.

صحة الاتجاه (RTL): يُعرض النص العربي من اليمين إلى اليسار بشكل صحيح، مع معالجة سليمة للنصوص ثنائية الاتجاه (عربي مختلط بإنجليزي وأرقام ومقاطع كود).

التشكيل (Shaping): ربط الحروف بأشكالها الصحيحة (بداية/وسط/نهاية/منفصلة) بدل عرضها متقطّعة كما في معظم أدوات الطرفية.

تحديد ونسخ منطقي: عند تحديد نص عربي معروض بصريًا، يُنسخ إلى الحافظة بترتيبه المنطقي الصحيح فيلصق سليمًا في أي مكان.

إدخال عربي: كتابة الأوامر والرسائل بالعربية داخل حقل الإدخال مع سلوك تحرير صحيح للاتجاه.

تعدّد مزوّدي الموديلات

OpenAI و Anthropic و Google و AWS Bedrock و Groq و Azure و OpenRouter و Copilot وغيرها عبر Models.dev، وموديلات محلية عبر LOCAL_ENDPOINT. بدّل الموديل حسب المهمة.

أوضاع Build / Plan

وكيلان تتنقّل بينهما بمفتاح Tab: build وصول كامل للتعديل والتنفيذ، وplan قراءة فقط يقترح خطة دون لمس الملفات. راجع الخطة قبل التنفيذ.

الوكلاء المخصّصة

أنشئ وكلاء عبر arabcode agent create بصلاحيات دقيقة (bash, read, edit, grep, webfetch…). فريق من الوكلاء المتخصّصين يعطي نتائج أدق لكل مجال.

ملف AGENTS.md و/init

أمر /init يحلّل المشروع وينشئ ملف AGENTS.md يوثّق التقنيات وأسلوب الكود والبنية، ويُقرأ تلقائيًا في كل جلسة. لا تكرّر شرح قواعد مشروعك.

لقطات الجلسة والتراجع

يحفظ arabcode لقطة لكل جلسة تسمح بالرجوع إلى أي رسالة سابقة، بما في ذلك استعادة تعديلات الملفات لحظة تلك النقطة — بدل التراجع خطوة بخطوة.

دعم MCP و Code Mode

إدارة كاملة لخوادم Model Context Protocol (محلية/بعيدة، OAuth، قوالب موارد)، وCode Mode لتشغيل سكربتات تنسيق مقيّدة فوق أدوات MCP بدل استدعائها واحدة تلو الأخرى.

YOLO Mode

وضع في الواجهة النصية يوافق تلقائيًا على كل طلبات الصلاحيات دون مقاطعتك — سرعة أكبر في المهام الموثوقة، يُستخدم بحذر في بيئات تثق فيها بالكود.

الأوامر المخصّصة

ملفات Markdown تتحوّل إلى أوامر جاهزة، وتدعم متغيّرات بصيغة $NAME تُعبّئ عند التشغيل. حوّل المهام المتكررة إلى أمر واحد.

أوضاع غير تفاعلية

arabcode run للسكربتات (مع JSON و--replay)، وarabcode serve خادم HTTP، وarabcode web واجهة ويب، و--mini، وattach، وخادم ACP.

تكامل GitHub / GitLab

arabcode github install يهيّئ workflow آليًا، وarabcode pr <رقم> يجلب فرع الـ PR ويشغّل عليه. مراجعة كود آلية داخل خط CI/CD.

تكامل LSP

دعم Language Server Protocol لذكاء برمجي حقيقي — اقتراحات دقيقة، معرفة الأنواع، تنقّل في الكود — وليس مجرّد تحليل نصي. فهم أعمق يقلّل الأخطاء.

إدارة الجلسات والتكلفة

عرض/حذف/تصدير الجلسات (JSON)، استنساخ workspace كامل، أمر stats لاستهلاك التوكنز والتكلفة، و/share لرابط مشاركة. شفافية كاملة على الإنفاق.

نظام Skills

اكتشاف تلقائي للـ skills والوكلاء المبنية على ملفات، مع حفظ المسارات كمسارات ملفات حقيقية. أعِد استخدام منطق متخصّص بدل إعادة شرحه.

التوافق مع .claude

يقرأ arabcode مجلد .claude تلقائيًا (بما فيه CLAUDE.md والـ skills)، ويُتحكَّم به عبر متغيّرات موروثة مثل OPENCODE_DISABLE_CLAUDE_CODE. استخدم أدواتك السابقة بأقل تعديل.

Plugins و SDK

نظام plugins قابل للتوسّع (hook APIs، واجهة V2 لـ Effect و Promise)، و SDK يعرض بثّ أحداث حيّ ووصولًا للجلسات النشطة وسجلًا مقسّمًا ونقاط صلاحيات. ابنِ فوق arabcode كمنصّة.

أربعة أسباب في سطور

  • العربية الكاملة: اتجاه RTL صحيح وتشكيل سليم وتحديد ونسخ منطقي وإدخال عربي — لا توفّره أدوات الطرفية الأخرى.
  • إرث opencode الناضج: منظومة ميزات مختبَرة على نطاق واسع دون إعادة اختراع العجلة.
  • الأتمتة الشاملة: أوضاع run/serve/web/ACP وتكامل GitHub تغطّي كل سيناريوهات السكربتات والإنتاج.
  • الاستقلالية عن المزوّد: لست مقيّدًا بموديل أو شركة واحدة — بدّل بين المزوّدين والموديلات المحلية بحرية.
