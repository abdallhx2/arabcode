القسم الأول

النماذج المجانية — كيف تبدأ دون بطاقة

هناك عدد كبير من الموديلات، لكن قليلًا منها يعمل جيّدًا كوكيل برمجة. أسهل بداية هي النماذج المجانية المتاحة عبر بوّابة الموديلات — تجرّب arabcode كاملًا دون أي تكلفة ولا بطاقة ائتمان.

نصيحة إن كنت جديدًا على مزوّدي الـ LLM، ابدأ بأحد النماذج المجانية أدناه. لا تحتاج إلى بطاقة، ويمكنك الترقية لاحقًا لموديل أقوى بأمر واحد.

كيف تعمل

  1. شغّل أمر الربط داخل الواجهة النصية (TUI)، واختر بوّابة النماذج المجانية:

    /connect

  2. سجّل الدخول والصق مفتاح الـ API عند الطلب. النماذج المجانية لا تتطلّب بطاقة أو رصيدًا.

    ┌ API key
│
│
└ enter

  3. اعرض قائمة الموديلات المتاحة واختر أحد المجّانية:

    /models

القسم الثاني

ربط الموديلات والمزوّدون

لإضافة مزوّد تحتاج خطوتين: أضف مفتاح الـ API عبر أمر الربط، ثم اضبط المزوّد في ملف الإعداد عند الحاجة. تُخزَّن المفاتيح محليًا في ملف auth.json ضمن مجلد بيانات arabcode.

عبر سطر الأوامر

يمكنك إدارة المصادقة من خارج الواجهة النصية أيضًا:

bash — إدارة المصادقة
arabcode auth login     # اختر مزوّدًا وأدخل مفتاح الـ API
arabcode auth list      # اعرض المزوّدين المسجّلين
arabcode auth logout    # احذف بيانات اعتماد مزوّد

OpenAI

أنشئ مفتاحًا من لوحة OpenAI، ثم شغّل /connect واختر OpenAI والصق المفتاح. أو استخدم اشتراك ChatGPT Plus/Pro كمزوّد مصادقة مباشر دون إعداد.

Anthropic (Claude)

شغّل /connect واختر Anthropic، ثم أدخِل مفتاح API. بعدها تظهر كل موديلات Claude في أمر /models.

Google (Gemini)

احصل على مفتاح من Google AI Studio، ثم /connect واختر Google والصق المفتاح. يدعم arabcode كذلك Vertex AI للاستخدام المؤسسي.

GitHub Copilot

استخدم اشتراك Copilot (Pro/Business/Enterprise) كمزوّد مصادقة مباشر — بلا مفتاح API منفصل.

ضبط المزوّد في ملف الإعداد

خصّص أي مزوّد عبر قسم provider في ملف إعداد arabcode. مثال: تغيير عنوان القاعدة (baseURL) لتوجيه الطلبات عبر وسيط أو نقطة نهاية مخصّصة.

opencode.json — عنوان قاعدة مخصّص (صيغة إعداد موروثة)
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "anthropic": {
      "options": {
        "baseURL": "https://api.anthropic.com/v1"
      }
    }
  }
}
اسم موروث صيغة ملف الإعداد ومخطّطه (opencode.json) موروثة من محرّك opencode الذي يبني عليه arabcode. احتفظ بها كما هي؛ فهي أسماء داخلية لا علامة تجارية.

إخفاء موديلات من القائمة

استخدم blacklist لإزالة موديلات من منتقي /models، أو whitelist لإبقاء المدرَجة فقط.

opencode.json — إبقاء موديل واحد فقط
{
  "provider": {
    "anthropic": {
      "whitelist": ["claude-sonnet-4-20250514"]
    }
  }
}
موديلات محلية

شغّل موديلاتك على جهازك (self-hosted)

يدعم arabcode أي خادم متوافق مع OpenAI — مثل Ollama و LM Studio و llama.cpp — عبر ضبط baseURL على نقطة النهاية المحلية، أو عبر متغيّر البيئة الموروث LOCAL_ENDPOINT.

opencode.json — موديل محلي عبر Ollama
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "ollama": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "Ollama (local)",
      "options": {
        "baseURL": "http://localhost:11434/v1"
      },
      "models": {
        "qwen3-coder": { "name": "Qwen3 Coder (local)" }
      }
    }
  }
}
الخادم المحلينقطة النهاية الافتراضيةحزمة AI SDK
Ollamahttp://localhost:11434/v1@ai-sdk/openai-compatible
LM Studiohttp://127.0.0.1:1234/v1@ai-sdk/openai-compatible
llama.cpp (llama-server)http://127.0.0.1:8080/v1@ai-sdk/openai-compatible
نصيحة إن لم تعمل استدعاءات الأدوات (tool calls) جيّدًا مع موديل محلي، اختر موديلًا بدعم قوي لاستدعاء الأدوات (مثل Qwen-Coder أو DeepSeek-Coder)، وزِد قيمة سياق النموذج (num_ctx) في Ollama إلى نحو 16k–32k.

LOCAL_ENDPOINT وOPENCODE_* أسماء متغيّرات بيئة موروثة من opencode — احتفظ بها كما هي.

الرئيسية النماذج الميزات التثبيت الصفحة التعريفية GitHub

arabcode يعرّب الطرفية بالكامل (اتجاه RTL + تشكيل عربي) — وهي إضافته الجوهرية فوق المحرّك. وهو مبني على opencode مفتوح المصدر — رخصة MIT. الميزات الموروثة هي من opencode؛ إسهام arabcode الأساسي هو طبقة العربية (الاتجاه والتشكيل والإدخال).
