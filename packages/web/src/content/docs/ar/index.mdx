[**arabcode**](/) هو وكيل برمجة بالذكاء الاصطناعي مفتوح المصدر، عربيّ أولًا — عرض RTL صحيح وتشكيل وحروف متصلة وتحديد منطقي، مبني على محرّك opencode مفتوح المصدر.

لنبدأ.

---

#### المتطلبات

لاستخدام arabcode في الطرفية، ستحتاج إلى:

1. محاكي طرفية حديث مثل:
   - [WezTerm](https://wezterm.org) متعدد المنصات
   - [Alacritty](https://alacritty.org) متعدد المنصات
   - [Ghostty](https://ghostty.org) على Linux وmacOS
   - [Kitty](https://sw.kovidgoyal.net/kitty/) على Linux وmacOS

2. مفاتيح API لمزوّدي نماذج اللغة (LLM) الذين تريد استخدامهم.

أسهل طريقة لتثبيت arabcode هي عبر سكربت التثبيت.

## الإعداد

مع arabcode يمكنك استخدام أي مزود LLM عبر إعداد مفاتيح API الخاصة به.

1. شغّل الأمر `/connect` في واجهة TUI واختر مزوّدا.

   ```txt
   /connect
   ```

2. سجّل الدخول لدى المزوّد، أو انسخ مفتاح API من لوحة التحكم الخاصة به.

3. الصق مفتاح API.

   ```txt
   ┌ API key
   │
   │
   └ enter
   ```

يمكنك الاختيار من بين أكثر من 75 مزوّدا. [اعرف المزيد](/docs/providers#directory).

## التهيئة

بعد أن قمت بإعداد مزوّد، يمكنك الانتقال إلى المشروع الذي تريد العمل عليه.

```bash
cd /path/to/project
```

ثم شغّل arabcode.

```bash
arabcode
```

بعد ذلك، هيّئ arabcode للمشروع عبر تشغيل الأمر التالي.

```bash frame="none"
/init
```

سيجعل هذا arabcode يحلل مشروعك ويُنشئ ملف `AGENTS.md` في جذر المشروع.

:::tip
يستحسن أن تقوم بعمل commit لملف `AGENTS.md` الخاص بمشروعك إلى Git.
:::

يساعد هذا arabcode على فهم بنية المشروع وأنماط الترميز المستخدمة.

## الاستخدام

أصبحت الآن جاهزا لاستخدام arabcode للعمل على مشروعك. لا تتردد في سؤاله أي شيء — بالعربية أو الإنجليزية!

إذا كنت جديدا على استخدام وكيل برمجة بالذكاء الاصطناعي، فإليك بعض الأمثلة التي قد تساعد.

---

### اطرح أسئلة

يمكنك أن تطلب من arabcode شرح قاعدة الشيفرة لك.

:::tip
استخدم المفتاح `@` للبحث التقريبي عن الملفات داخل المشروع.
:::

```txt frame="none" "@packages/functions/src/api/index.ts"
اشرح لي كيف تُدار المصادقة في @packages/functions/src/api/index.ts
```

يفيد هذا عندما تكون هناك أجزاء من قاعدة الشيفرة لم تعمل عليها.

---

### أضف ميزات

يمكنك أن تطلب من arabcode إضافة ميزات جديدة إلى مشروعك، لكننا نوصي أولا بطلب إنشاء خطة.

1. **أنشئ خطة**

   لدى arabcode _وضع Plan_ يعطل قدرته على إجراء التغييرات، ويقترح بدلا من ذلك _كيف_ سينفّذ الميزة.

   انتقل إليه باستخدام مفتاح **Tab**. سترى مؤشرا لذلك في الزاوية السفلية.

   ```bash frame="none" title="التبديل إلى وضع Plan"
   <TAB>
   ```

   الآن لنصف ما نريده أن يفعله.

   ```txt frame="none"
   عند حذف المستخدم لملاحظة، نريد وضع علامة "محذوفة" عليها في قاعدة البيانات.
   ثم أنشئ شاشة تعرض كل الملاحظات المحذوفة مؤخرا.
   من هذه الشاشة يمكن للمستخدم استعادة الملاحظة أو حذفها نهائيا.
   ```

   احرص على تزويد arabcode بتفاصيل كافية ليفهم ما تريد. يساعد أن تتحدث إليه كما لو كنت تتحدث إلى مطور مبتدئ ضمن فريقك.

   :::tip
   أعطِ arabcode سياقا وأمثلة كافية لمساعدته على فهم ما تريد.
   :::

2. **حسّن الخطة**

   بعد أن يقدم لك خطة، يمكنك إعطاؤه ملاحظات أو إضافة تفاصيل أكثر.

   ```txt frame="none"
   نريد تصميم هذه الشاشة الجديدة اعتمادا على تصميم استخدمته من قبل.
   [Image #1] ألقِ نظرة على هذه الصورة واستخدمها كمرجع.
   ```

   :::tip
   اسحب الصور وأفلِتها داخل الطرفية لإضافتها إلى الطلب.
   :::

   يستطيع arabcode فحص أي صور تزوده بها وإضافتها إلى الطلب. يمكنك فعل ذلك عبر سحب الصورة وإفلاتها داخل الطرفية.

3. **ابنِ الميزة**

   عندما تشعر بالارتياح للخطة، عد إلى _وضع Build_ بالضغط على مفتاح **Tab** مرة أخرى.

   ```bash frame="none"
   <TAB>
   ```

   ثم اطلب منه تنفيذ التغييرات.

   ```txt frame="none"
   ممتاز! نفّذ التغييرات.
   ```

---

### نفّذ تغييرات

للتغييرات الأبسط، يمكنك أن تطلب من arabcode تنفيذها مباشرة دون الحاجة لمراجعة خطة أولا.

```txt frame="none" "@packages/functions/src/settings.ts" "@packages/functions/src/notes.ts"
نحتاج إضافة مصادقة إلى المسار ‎/settings‎. انظر كيف تُدار في المسار ‎/notes‎
في @packages/functions/src/notes.ts وطبّق المنطق نفسه
في @packages/functions/src/settings.ts
```

تأكد من تقديم قدر مناسب من التفاصيل لكي يجري arabcode التغييرات الصحيحة.

---

### تراجع عن التغييرات

لنفترض أنك طلبت من arabcode إجراء بعض التغييرات.

```txt frame="none" "@packages/functions/src/api/index.ts"
هل يمكنك إعادة هيكلة الدالة في @packages/functions/src/api/index.ts؟
```

لكن تتضح لك لاحقا أنها ليست ما تريده. يمكنك **التراجع** عن التغييرات باستخدام الأمر `/undo`.

```bash frame="none"
/undo
```

سيقوم arabcode الآن بعكس التغييرات التي أُجريت ويعرض رسالتك الأصلية مجددا.

```txt frame="none" "@packages/functions/src/api/index.ts"
هل يمكنك إعادة هيكلة الدالة في @packages/functions/src/api/index.ts؟
```

من هنا يمكنك تعديل الطلب وطلب المحاولة مرة أخرى.

:::tip
يمكنك تشغيل `/undo` عدة مرات للتراجع عن عدة تغييرات.
:::

أو يمكنك **إعادة تنفيذ** التغييرات باستخدام الأمر `/redo`.

```bash frame="none"
/redo
```
