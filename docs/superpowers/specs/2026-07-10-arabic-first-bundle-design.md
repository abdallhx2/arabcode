# arabcode — تصميم حزمة «العربية أولاً»

**التاريخ:** 2026-07-10
**الفرع:** `arabcode`
**يكمل:** سلسلة تصاميم RTL (العرض، الإدخال، Markdown، التحديد والنسخ)

## المشكلة

ثلاثة احتكاكات يومية يواجهها المبرمج العربي ولا تعالجها أي أداة ترميز طرفية:

1. **التخطيط الخاطئ للوحة المفاتيح:** يكتب `/اثمح` وهو يقصد `/help` (نسي التبديل للإنجليزية)، أو `hgsghl` وهو يقصد «السلام» (نسي التبديل للعربية). اليوم: الأمر يفشل صامتاً أو يخرج نص مهمل.
2. **الأوامر إنجليزية فقط:** أسماء أوامر الشرطة المائلة (`/new`, `/help`, ...) إنجليزية، رغم أن الواجهة معرّبة بالكامل.
3. **خلط اللغات في المخرجات:** المستخدم يريد الحوار والشرح بالعربية، لكن الكود والتعليقات ورسائل commit يجب أن تبقى إنجليزية (مشاريع مفتوحة المصدر، فرق مختلطة). اليوم يجب طلب ذلك يدوياً في كل جلسة.

## القرارات (موافقة المستخدم)

- **سلوك المصحّح: هجين ذكي** — أوامر `/` تُصحَّح تلقائياً وفوراً (لا لبس: الأوامر مجموعة معروفة)؛ النثر اقتراح غير مزعج يُقبل بمفتاح (التصحيح التلقائي للنثر قد يخطئ).
- **الأوامر العربية: ثنائية في الإكمال** — كل أمر له اسمان يعملان دائماً؛ القائمة تعرض الاسم العربي وبجانبه الإنجليزي بلون خافت؛ الكتابة بأيهما تطابق.
- **حدود اللغة: الملفات إنجليزية** — الشرح والحوار بالعربية؛ كل ما يُكتب في ملفات المشروع (كود، تعليقات، معرّفات، رسائل commit، توثيق) بالإنجليزية.

## المعمارية

### المكوّن 1: مصحّح تخطيط لوحة المفاتيح

#### الوحدة الأساسية — `packages/tui/src/util/layout-fix.ts` (جديد)

- **جدولا تحويل ثابتان** لتخطيط عربي 101 القياسي ↔ QWERTY: الطبقة الأساسية كاملة (`ض↔q`, `ص↔w`, `ث↔e`, ..., `ذ↔\``) + مفاتيح الترقيم المزدوجة (`و↔,`, `ز↔.`, `ظ↔/`, `ك↔;`, `ط↔'`, `ج↔[`, `د↔]`, `لا↔b`, `ى↔n`, `ة↔m`, `ء↔x`, `ؤ↔c`, `ر↔v`, `ئ↔z`). ملاحظة `لا` ↔ `b`: حرفان منطقيان لمفتاح واحد — التحويل عربي→لاتيني يعالجها كثنائية `لا`، والعكس ينتجها كثنائية.
- `mapArToEn(text: string): string` و`mapEnToAr(text: string): string` — تحويل حرفياً بالجدول؛ الأحرف خارج الجدول تمرّ كما هي.
- `looksLikeArTypedAsEn(word: string): boolean` — كشف محافظ للاتيني المقصود عربياً: طول ≥ 4، كل أحرفه في الجدول، **نسبة حروف العلة اللاتينية (aeiou) < 15%** (الحروف العربية الشائعة ا/ل/م/ن/ت تُنتج h/g/l/k/j — صوامت، فالهراء الناتج فقير بحروف العلة بينما الإنجليزية الحقيقية غنية بها)، وليس ضمن قائمة استثناء مدمجة صغيرة للكلمات التقنية عديمة العلّة الشائعة (`npm`, `js`, `ts`, `git`, `ssh`, `sql`, `css`, `html`, `xml`, `json`, `grpc`, `cmd`, `ctrl`, ...).
- `looksLikeEnTypedAsAr(word: string): boolean` — كشف العربي المقصود إنجليزياً: كل أحرفه عربية، وناتج `mapArToEn` موجود في **قائمة كلمات إنجليزية شائعة مدمجة** (مئات الكلمات الأكثر تداولاً في المحادثة التقنية، ثابت في الملف) — بلا قاموس لا يمكن التمييز لأن أغلب تسلسلات الحروف العربية سليمة شكلاً.
- `fixWord(word: string): { fixed: string; direction: "ar→en" | "en→ar" } | undefined` — يجمع الكشف والتحويل.

#### تصحيح أوامر `/` الفوري — `packages/tui/src/component/prompt/autocomplete.tsx`

في مسار `onInput` (حيث يُكشف `/` في الموضع 0):

1. إذا كان النص بعد `/` **عربياً** ولا يطابق (بادئةً) أيَّ `slashName` أو alias عربي معروف → حوّله بـ`mapArToEn`؛ إن طابق الناتج بادئة أمر معروف → **استبدل نص الحقل فوراً** (نمط `deleteRange` + `insertText` المستخدم أصلاً في الملف) واستمرّ بالإكمال الطبيعي.
2. والعكس: نص **لاتيني** لا يطابق شيئاً لكن `mapEnToAr` له يطابق اسماً عربياً → استبدال فوري.
3. المطابقة تتم ضد قائمة الأوامر الحيّة (`useCommandSlashes()` + أسماء المكوّن 2) — لا قائمة مكرّرة.

لا تصحيح إن طابق النصُّ الأصلي شيئاً (لا نفسد إدخالاً صحيحاً)، ولا تصحيح إن لم يطابق الناتج شيئاً (لا نخمّن).

#### اقتراح النثر — `packages/tui/src/component/prompt/index.tsx`

- عند إتمام كلمة (كتابة مسافة)، تُفحص **الكلمة المكتملة للتو** بـ`fixWord`. أيُّ كلمة مكتشفة تُضاف لمجموعة الكلمات المرشّحة ويُعرض **شريط تلميح** فوق حقل الإدخال بنمط خافت يضم ناتج تحويلها: `هل تقصد: السلام؟ ‹Tab›` (الكشف المحافظ في `fixWord` هو صمّام عدم الإزعاج — لا حاجة لعتبات إضافية).
- **Tab** يقبل: يستبدل كل الكلمات المرشّحة الحالية بناتج تحويلها في الحقل ويخفي التلميح. **الكتابة** تُبقي التلميح محدثاً (وتخفيه إن زالت المرشّحات بالتحرير). **Escape** يرفضه لمحتوى السطر الحالي. **Enter** يُرسل النص كما هو دون تطبيق التلميح (التلميح سلبي لا يعترض الإرسال).
- Tab هنا لا يتعارض مع autocomplete: التلميح لا يظهر إلا حين لا توجد قائمة إكمال مفتوحة (قائمة الإكمال لها الأولوية).

### المكوّن 2: الأوامر العربية الثنائية

#### جدول الأسماء — `packages/tui/src/util/commands-ar.ts` (جديد)

`export const COMMAND_AR: Record<string, string>` — من `slashName` الإنجليزي إلى الاسم العربي:

| إنجليزي | عربي | إنجليزي | عربي |
|---|---|---|---|
| help | مساعدة | new | جديد |
| sessions | جلسات | models | نماذج |
| agents | وكلاء | themes | سمات |
| connect | اتصال | exit | خروج |
| share | مشاركة | unshare | إلغاء-المشاركة |
| compact | ضغط | undo | تراجع |
| redo | إعادة | thinking | تفكير |
| export | تصدير | editor | محرر |
| skills | مهارات | status | حالة |
| mcps | خوادم | variants | تنويعات |
| workspaces | مساحات | org | منظمة |
| debug | تنقيح | diff | فروقات |
| move | نقل | warp | وورب |

(القائمة النهائية تُستخرج من كل تسجيلات `useBindings` وقت التنفيذ؛ الأوامر المخصصة وأوامر الخوادم خارج النسخة الأولى.)

#### الحقن المركزي — `packages/tui/src/keymap.tsx` (`useCommandSlashes()`)

نقطة واحدة تغذّي إكمال `/` كله. لكل مدخلة لها `slashName` وله ترجمة في الجدول:

- `aliases`: يُضاف `"/" + arName` إلى القائمة الموجودة → **الكتابة بالعربي تطابق** (fuzzysort يفتّش مفتاح aliases أصلاً).
- `display`: يصبح `"/" + arName` (الاسم العربي هو الأساسي).
- `description`: يصبح `` `${desc} · /${slashName}` `` → الوصف العربي الحالي + **الاسم الإنجليزي القانوني بخط خافت** (حقل description يُرسم خافتاً `textMuted` أصلاً).
- أمرٌ بلا ترجمة يبقى كما هو تماماً.

التنفيذ عند الاختيار يمرّ بـ`dispatchCommand(entry.command.name)` الموجود — لا تغيير في مسار التنفيذ. لوحة الأوامر (`command-palette.tsx`) تعرض العناوين العربية أصلاً — لا تغيير.

### المكوّن 3: الوضع ثنائي اللغة

#### التوجيه — `packages/opencode/src/session/bilingual.ts` (جديد)

وحدة تصدّر نصّ توجيه ثابتاً (بالإنجليزية — لغة أوثق التزاماً للنماذج، والسلوك الناتج عربي):

> Conversational output policy (arabcode): Address the user in Arabic — explanations, summaries, questions, and progress notes. Everything written INTO project files stays in English: code, identifiers, comments, commit messages, PR descriptions, and documentation files. Technical terms may remain in English inside Arabic prose. If the user writes in another language, mirror their language instead.

#### الحقن — `packages/opencode/src/session/prompt.ts`

في نقطة تجميع `system` الحالية (حيث تُضم `env` + `instructions` + `mcp` + `skills`): يُضاف نص `bilingual.ts` بعد `instructions` عندما يكون الوضع مفعّلاً.

#### الإعداد — `packages/core/src/config.ts`

مفتاح جديد في `Config.Info`:

```
arabcode: Schema.Struct({
  bilingual: Schema.optional(Schema.Boolean),  // الافتراضي: true
}).pipe(Schema.optional)
```

- مفعّل افتراضياً (`undefined` → true). التعطيل: `{"arabcode": {"bilingual": false}}` أو متغيّر البيئة `ARABCODE_BILINGUAL=false` (نمط `truthy` في `flag.ts` مع اعتبار غياب المتغيّر تفعيلاً — نفس نمط `OPENCODE_EXPERIMENTAL_DISABLE_COPY_ON_SELECT` المقلوب).
- أولوية: متغيّر البيئة > الإعداد > الافتراضي.

## حالات الحافة

- **`/` + نص يطابق أمراً عربياً وإنجليزياً معاً بعد التحويل:** لا يحدث عملياً (المجموعتان منفصلتا الحروف)؛ إن حدث تُفضَّل المطابقة بلا تحويل.
- **كلمة نثر صحيحة في اللغتين** (مثل `she` ↔ ناتج عربي صحيح): الكشف يتطلب هراءً في لغة المصدر — كلمة إنجليزية حقيقية (في قائمة الكلمات أو غنية بالعلّة) لا تُرشَّح أبداً.
- **مقاطع كود داخل النثر** (backticks، مسارات، أسماء متغيرات): تُستثنى من فحص النثر — أي كلمة تحوي `/`, `.`, `_`, `-`, أرقاماً، أو داخل backticks لا تُفحص.
- **RTL في شريط التلميح وقائمة الإكمال:** النصوص العربية الجديدة تمرّ عبر خط أنابيب RTL القائم (TextRenderable مغطّى بالخطافات) — لا عمل إضافي، لكن تُختبر بصرياً.
- **الوضع ثنائي اللغة مع AGENTS.md يفرض لغة أخرى:** توجيه المستخدم في `AGENTS.md` يأتي في `instructions` وتوجيهنا بعده؛ الجملة الأخيرة (mirror their language) + كون توجيه المستخدم أكثر تحديداً يحسمان للنموذج. لا منطق خاص.
- **الوكلاء الداخليون** (compaction/title/summary): التوجيه يُحقن في مسار جلسة المحادثة الرئيسي فقط؛ عناوين الجلسات تبقى بلغة المحتوى.

## الاختبار

- **وحدة `layout-fix`:** تحويلات مزدوجة الاتجاه (`السلام`↔`hgsghl`، `اثمح`↔`help`، `لا`↔`b`)؛ عتبات الكشف (كلمة إنجليزية حقيقية لا تُكشف، `npm` مستثناة، هراء عديم العلّة يُكشف)؛ `fixWord` بالاتجاهين؛ كلمات بمحارف مختلطة تمرّ سليمة.
- **وحدة `commands-ar`:** كل مفتاح في الجدول له قيمة عربية غير فارغة وفريدة.
- **تكامل الإكمال:** `useCommandSlashes` بعد الحقن — مدخلة `new` تحمل alias `/جديد`، display عربي، description يحوي `/new`؛ الكتابة `/جديد` تُطابق عبر fuzzysort؛ أمر بلا ترجمة سليم بلا تغيير.
- **تكامل التصحيح الفوري:** إدخال `/اثمح` في autocomplete يستبدل النص بـ`/help`؛ `/help` الصحيح لا يُمسّ؛ هراء لا يطابق شيئاً لا يُمسّ.
- **تكامل ثنائي اللغة:** بناء `system` لجلسة يحوي نص التوجيه حين مفعّل ولا يحويه حين `bilingual: false` أو `ARABCODE_BILINGUAL=false`.
- **عدم انحدار:** كل `test/util/rtl*` تمرّ؛ typecheck نظيف.
- **تحقق فعلي:** تشغيل التطبيق — كتابة `/اثمح` تتصحّح فورياً؛ كتابة `hgsghl` تُظهر تلميح «هل تقصد: السلام؟» وTab يقبله؛ `/جديد` يفتح جلسة؛ رد المساعد عربي الشرح إنجليزي الكود.

## خارج النطاق (النسخة الأولى)

- أسماء عربية للأوامر المخصصة وأوامر خوادم MCP.
- تصحيح التخطيط داخل نصوص طويلة ملصوقة (اللصق لا يُفحص — الكتابة الحية فقط).
- طبقة Shift الكاملة لتخطيط 101 (الأساسية + الترقيم المزدوج تكفي للأوامر والنثر).
- تخطيطات عربية غير قياسية (مصري/مغاربي معدّل).
- ترجمة توجيه ثنائي اللغة حسب لهجات.

## معايير النجاح

- `/اثمح` تتحوّل `/help` فورياً دون أي تدخّل؛ `/help` الصحيحة لا تتأثر إطلاقاً.
- `hgsghl` تُظهر اقتراحاً غير مزعج يُقبل بـTab ويُرفض بالتجاهل.
- `/جديد` و`/new` كلاهما يعمل، والقائمة تعرض العربي أولاً والإنجليزي خافتاً.
- ردود المساعد: شرح عربي، وكل محتوى الملفات ورسائل commit إنجليزي — دون طلب يدوي، وقابل للتعطيل بإعداد واحد.
- صفر انحدار في اختبارات RTL والـtypecheck.
