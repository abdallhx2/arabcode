# مسرد المصطلحات — arabcode

مرجع إلزامي لكل الترجمات. المصطلح الإنجليزي يُترجم **دائماً** بنفس الكلمة العربية.

## قواعد عامة

1. **يُترجم فقط** النص المرئي للمستخدم: عناوين، تسميات أزرار، رسائل، أوصاف أوامر، رسائل خطأ، نصوص placeholder.
2. **لا يُترجم أبداً**: مفاتيح الكائنات (keys)، المعرّفات (ids)، قيم enum، أسماء الأحداث، المسارات، الروابط، أسماء الحزم `@opencode-ai/*`، أسماء الملفات، اختصارات لوحة المفاتيح (مثل `ctrl+x`)، أسماء المتغيرات والدوال، محتوى ملفات الاختبار `.test.ts`، النصوص داخل أوامر bash/git.
3. مصطلحات git تبقى إنجليزية كما هي: commit, branch, stash, rebase, merge, PR, diff (في السياق التقني المباشر). عند وصفها بجملة تُستخدم الترجمة من الجدول.
4. أسماء العلم تبقى كما هي: GitHub, Anthropic, OpenAI، أسماء النماذج والمزودين.
5. اسم البرنامج المعروض: **arabic code** — واسم الأمر: **arabcode**. لا تُستخدم كلمة opencode في أي نص مرئي.
6. الأرقام تبقى أرقاماً غربية (1, 2, 3).
7. صيغة الجارية: «جارٍ + المصدر» (جارٍ التحميل…). صيغة الفشل: «فشل + المصدر» (فشل حذف الجلسة).
