# دليل ميزات arabcode

arabcode هو أول أداة سطر أوامر (CLI) للذكاء الاصطناعي في الطرفية بالعربية الكاملة — باتجاه RTL صحيح وتشكيل عربي سليم. وهو مبني على محرّك **opencode** مفتوح المصدر، ولذلك يرث منظومة ميزات ناضجة ومختبَرة على نطاق واسع، ويضيف فوقها ما يميّزه فعليًا: عرضٌ صحيح للعربية داخل الطرفية لا توفّره أي أداة أخرى من نوعه. باختصار، أنت تحصل على قوة أداة برمجة حديثة بالكامل، لكن بلغة عربية تُقرأ وتُكتب كما ينبغي.

> ملاحظة عن النسب: arabcode يعتمد على opencode ويضيف إليه دعم العربية؛ الميزات الموروثة هي من opencode، وإسهام arabcode الأساسي هو طبقة العربية (الاتجاه والتشكيل والإدخال) المطبّقة فعليًا في هذا المستودع.

---

## 1. دعم العربية الكامل (RTL + تشكيل) — ميزة arabcode المميِّزة

هذه هي الإضافة الحقيقية التي يقدّمها arabcode فوق opencode، وهي سبب وجوده.

- **صحة الاتجاه (RTL):** يُعرض النص العربي من اليمين إلى اليسار بشكل صحيح داخل الطرفية، مع معالجة سليمة للنصوص ثنائية الاتجاه (عربي مختلط بإنجليزي وأرقام ومقاطع كود).
- **التشكيل (Shaping):** ربط الحروف العربية بأشكالها الصحيحة (بداية/وسط/نهاية/منفصلة) بدل عرضها متقطّعة كما يحدث في معظم أدوات الطرفية.
- **تحديد ونسخ منطقي:** عند تحديد نص عربي معروض بصريًا، يُنسخ إلى الحافظة بترتيبه **المنطقي** الصحيح، فيلصق سليمًا في أي مكان آخر.
- **إدخال عربي:** كتابة الأوامر والرسائل بالعربية داخل حقل الإدخال مع سلوك تحرير صحيح للاتجاه.
- **الفائدة:** تجربة عربية أصيلة في الطرفية — تقرأ ردود الوكيل وتكتب طلباتك بالعربية دون حروف مقطّعة ولا اتجاه معكوس.

---

## 2. تعدد مزودي الموديلات (Multi-Provider)

**الشرح:** arabcode مبني حول قائمة مزودين ضخمة عبر Models.dev — يدعم OpenAI و Anthropic Claude و Google Gemini و AWS Bedrock و Groq و Azure OpenAI و OpenRouter و GitHub Copilot و Cohere North و MiniMax و GLM و DigitalOcean و Snowflake Cortex وغيرها. كما يدعم موديلات محلية self-hosted متوافقة مع OpenAI عبر متغير `LOCAL_ENDPOINT` (اسم موروث من opencode).

**الفائدة:** بدّل الموديل حسب المهمة — موديل رخيص للمهام البسيطة وآخر قوي للتعقيد — دون تغيير الأداة، واربط موديلاتك المحلية بنفس الواجهة. لست مقيّدًا بمزوّد واحد.

---

## 3. أوضاع Build / Plan

**الشرح:** وكيلان مدمجان تتنقل بينهما بمفتاح `Tab`:

- `build`: وصول كامل للتعديل والتنفيذ (الوضع الافتراضي).
- `plan`: قراءة فقط، يقترح خطة ويحلّل الكود دون لمس الملفات، ويطلب الإذن قبل تشغيل أوامر bash.

**الفائدة:** راجع الخطة قبل التنفيذ لتقليل التعديلات الخاطئة على كود حسّاس أو إنتاجي، وهو مثالي لاستكشاف قواعد كود غير مألوفة.

---

## 4. الوكلاء المخصصة (Custom Agents)

**الشرح:** أنشئ وكلاء عبر `arabcode agent create` بصلاحيات دقيقة (bash, read, edit, glob, grep, webfetch, task, websearch...). كل وكيل يُعرّف بملف Markdown يحوي `YAML frontmatter` للإعدادات وجسم النص كـ system prompt. الوكلاء إما `primary` (يقود المحادثة) أو `subagent` (يُستدعى للمهام المتخصصة).

**الفائدة:** فريق من الوكلاء المتخصصين (مراجع أمان، كاتب توثيق، خبير اختبارات) يعطي نتائج أدق لكل مجال بدل وكيل عام واحد.

---

## 5. ملف AGENTS.md وأمر `/init`

**الشرح:** أمر `/init` يحلّل المشروع وينشئ ملف `AGENTS.md` في الجذر يوثّق التقنيات وأسلوب الكود والبنية وأوامر التطوير الشائعة، ويُقرأ تلقائيًا في كل جلسة (وهو مكافئ لملف CLAUDE.md).

**الفائدة:** لا تحتاج إلى تذكير الوكيل بقواعد مشروعك كل مرة — يتوافق تعامله مع الكود مع اصطلاحاتك مباشرة.

---

## 6. Session Snapshots والتراجع

**الشرح:** يحفظ arabcode لقطة لكل جلسة تسمح بالرجوع إلى أي رسالة سابقة، بما في ذلك استعادة التعديلات على الملفات لحظة تلك النقطة.

**الفائدة:** إذا سلك الوكيل مسارًا خاطئًا في تعديل معقّد، ترجع إلى نقطة محددة بدل التراجع خطوة بخطوة أو فقدان العمل بالكامل.

---

## 7. دعم MCP و Code Mode

**الشرح:** إدارة كاملة لخوادم Model Context Protocol — إضافة محلية أو بعيدة، عرض حالة الاتصال، مصادقة OAuth، قوالب موارد (resource templates)، وأدوات قراءة الموارد. ويوفّر **Code Mode MCP Adapter** تشغيل سكربتات orchestration مقيّدة فوق أدوات MCP المتصلة بدل استدعائها واحدة تلو الأخرى.

**الفائدة:** تكامل مباشر مع أدوات خارجية (قواعد بيانات، APIs، أنظمة ملفات) وتنسيق منطق معقّد بينها بكفاءة أعلى.

---

## 8. YOLO Mode

**الشرح:** وضع في الواجهة النصية (TUI) يوافق تلقائيًا على كل طلبات الصلاحيات دون مقاطعتك في كل خطوة.

**الفائدة:** سرعة أكبر في المهام الطويلة الموثوقة على حساب طبقة الأمان اليدوية — يُستخدم بحذر في بيئات تثق فيها بالكود.

---

## 9. الأوامر المخصصة (Custom Commands)

**الشرح:** ملفات Markdown في مجلدات محددة تتحوّل إلى أوامر جاهزة، وتدعم متغيرات مسمّاة بصيغة `$NAME` يُطلب منك تعبئتها عند التشغيل.

**الفائدة:** حوّل المهام المتكررة (فحص PR، تجميع سياق issue) إلى أمر واحد بدل كتابة نفس التعليمات كل مرة.

---

## 10. أوضاع التشغيل غير التفاعلية (للأتمتة)

- **`arabcode run "..."`**: تنفيذ غير تفاعلي مباشر مناسب للسكربتات، يدعم إخراج JSON خام و`--replay` لإعادة تشغيل الجلسة.
- **`arabcode serve`**: خادم HTTP بدون واجهة TUI للوصول البرمجي الكامل.
- **`arabcode web`**: نفس الخادم مع واجهة ويب تُفتح تلقائيًا.
- **`arabcode --mini`**: وضع CLI مصغّر.
- **`attach`**: ربط طرفية TUI بخادم backend يعمل عن بُعد.
- **خادم ACP**: تواصل عبر stdin/stdout بصيغة nd-JSON لتكاملات الأدوات المتقدمة.

**الفائدة:** تغطية كاملة لسيناريوهات الأتمتة — من سكربت bash بسيط إلى خادم API إنتاجي — مع تفادي "cold boot" لخوادم MCP بالاتصال بجلسة `serve` قائمة بدل تشغيل جديد كل مرة.

---

## 11. تكامل GitHub / GitLab

**الشرح:** أمر `arabcode github install` يهيّئ GitHub Actions workflow تلقائيًا، و`arabcode github run` يشغّل الوكيل داخل CI لمراجعات PR الآلية. وأمر `arabcode pr <رقم>` يجلب فرع الـ PR ويشغّل عليه مباشرة. كما يمكن استخدام اشتراك GitHub Copilot (Pro/Business/Enterprise) كمزود مصادقة مباشر.

**الفائدة:** مراجعة كود آلية داخل خط CI/CD، واستخدام اشتراك Copilot الموجود عندك بدل مفتاح API منفصل.

---

## 12. تكامل LSP

**الشرح:** دعم Language Server Protocol لذكاء برمجي حقيقي (اقتراحات دقيقة، معرفة الأنواع، تنقّل في الكود) وليس مجرد تحليل نصي.

**الفائدة:** فهم أعمق لبنية الكود يقلّل الأخطاء الناتجة عن تخمين النوع أو المرجع.

---

## 13. إدارة الجلسات والتكلفة

**الشرح:** عرض/حذف/تصدير الجلسات (JSON)، نقل جلسة بين المجلدات، استنساخ workspace كامل مع الملفات غير المرفوعة لـ git، أمر `stats` لعرض استهلاك التوكنز والتكلفة، وأمر `/share` لإنشاء رابط مشاركة.

**الفائدة:** شفافية كاملة على التكلفة والاستخدام — مهم إذا كنت تدير عدة مشاريع أو عملاء وتحتاج إلى تتبّع الإنفاق لكل واحد.

---

## 14. نظام Skills

**الشرح:** اكتشاف تلقائي للـ skills والوكلاء المبنية على ملفات، مع تخزين المسارات كمسارات ملفات حقيقية.

**الفائدة:** إعادة استخدام منطق متخصص محفوظ مسبقًا بدل إعادة شرحه في كل جلسة.

---

## 15. التوافق مع بنية `.claude`

**الشرح:** يقرأ arabcode مجلد `.claude` تلقائيًا إن وُجد في المشروع (بما فيه CLAUDE.md والـ skills)، ويمكن التحكم بذلك عبر متغيرات بيئة موروثة مثل `OPENCODE_DISABLE_CLAUDE_CODE` و`OPENCODE_DISABLE_CLAUDE_CODE_SKILLS` (أسماء داخلية موروثة من opencode).

**الفائدة:** إذا كانت لديك skills أو ملف CLAUDE.md مبنية مسبقًا، فتقدر تستخدمها في arabcode بأقل تعديل.

---

## 16. Plugins و SDK

**الشرح:** نظام plugins قابل للتوسّع (hook APIs مسمّاة، وواجهة plugin V2 لـ Effect و Promise)، و SDK كامل يعرض: بثّ أحداث حيّ (live event stream)، وصولًا للجلسات النشطة، سجل جلسات مقسّم صفحات (paged durable history)، ونقاط نهاية لطلبات الصلاحيات.

**الفائدة:** ابنِ أدوات وتكاملات مخصصة فوق arabcode كمنصة، وليس فقط استخدامه كتطبيق جاهز.

---

## 17. التثبيت والمنصات

يدعم arabcode التثبيت عبر npm و curl script و Homebrew ومن المصدر، على ماك ولينكس وويندوز (ويندوز و PowerShell مدعومان دعمًا كاملًا).

للأوامر الكاملة وخطوات التحقق وإزالة التثبيت، انظر [دليل التثبيت](INSTALL.ar.md).

---

## تُحدَّث باستمرار

بما أن arabcode مبني على opencode ذي وتيرة التطوير النشطة، فإن منظومة الميزات تتوسّع باستمرار. لهذا لا نثبّت في هذا الدليل أرقام إصدارات محددة لكل ميزة؛ اعتبر هذه القائمة صورة حيّة للقدرات المتاحة، وراجع مستودع المشروع لآخر المستجدات.

---

## لماذا arabcode؟

- **العربية الكاملة:** اتجاه RTL صحيح وتشكيل سليم وتحديد ونسخ منطقي وإدخال عربي — لا توفّره أدوات الطرفية الأخرى.
- **إرث opencode الناضج:** منظومة ميزات مختبَرة على نطاق واسع دون إعادة اختراع العجلة.
- **الأتمتة الشاملة:** أوضاع run/serve/web/ACP وتكامل GitHub تغطي كل سيناريوهات السكربتات والإنتاج.
- **الاستقلالية عن المزوّد:** لست مقيّدًا بموديل أو شركة واحدة — بدّل بين المزوّدين والموديلات المحلية بحرية.
